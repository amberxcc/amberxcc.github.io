什么是 Nginx？

Nginx (engine x) 是一款轻量级的 Web 服务器 、反向代理服务器及电子邮件（IMAP/POP3）代理服务器。

什么是反向代理？

反向代理（Reverse Proxy）方式是指以代理服务器来接受 internet 上的连接请求，然后将请求转发给内部网络上的服务器，并将从服务器上得到的结果返回给 internet 上请求连接的客户端，此时代理服务器对外就表现为一个反向代理服务器。


# 1. Nginx 安装

# 1.1 ubuntu-apt 安装

$ sudo apt-get update
$ sudo apt-get install nginx

常用命令：

$ nginx -s stop       # 快速关闭Nginx，可能不保存相关信息，并迅速终止web服务。
$ nginx -s quit       # 平稳关闭Nginx，保存相关信息，有安排的结束web服务。
$ nginx -s reload     # 因改变了Nginx相关配置，需要重新加载配置而重载。
$ nginx -s reopen     # 重新打开日志文件。
$ nginx -c filename   # 为 Nginx 指定一个配置文件，来代替缺省的。
$ nginx -t            # 不运行，仅仅测试配置文件。nginx 将检查配置文件的语法的正确性，并尝试打开配置文件中所引用到的文件。
$ nginx -v            # 显示 nginx 的版本。
$ nginx -V            # 显示 nginx 的版本，编译器版本和配置参数。

# 1.2 Docker 安装

$ docker run -d --name nginx --restart=always \
	-p 80:80 -p 443:443 \
	-v /docker/nginx/logs:/var/log/nginx \
    -v /docker/nginx/nginx.conf:/etc/nginx/nginx.conf \
	-v /docker/nginx/conf.d:/etc/nginx/conf.d \
	-v /docker/nginx/cert:/etc/nginx/cert \
    -v /www:/www \
	nginx

# 此处映射出来的文件为：日志目录、主配置文件、子配置目录、ssl证书目录、静态文件目录

💡 如果使用映射文件启动 nginx docker，而映射的文件没有配置好，可能启动不成功。

常用命令：

# 进入容器执行nginx命令
docker exec -it nginx -t
docker exec -it nginx -s reload
...

# 常用docker命令
docker restart nginx # 重启docker进程
...

# 2. Nginx 配置文件结构

  • main：全局配置部分用来配置对整个 server 都有效的参数。主要会设置一些影响 nginx 服务器整体运行的配置指令，主要包括配置运行 Nginx 服务器的用户（组）、允许生成的 worker process 数，进程 PID 存放路径、日志存放路径和类型以 及配置文件的引入等。

  • events：events 块涉及的指令主要影响 Nginx 服务器与用户的网络连接，常用的设置包括是否开启对多 worker process 下的网络连接进行序列化，是否允许同时接收多个网络连接，选取哪种事件驱动模型来处理连接请求，每个 worker process 可以同时支持的最大连接数等。

  • http：http 全局块配置的指令包括文件引入、MIME-TYPE 定义、日志自定义、连接超时时间、单链接请求数上限等。


# 3. 常见应用的配置

配置文件统一放在/etc/nginx/conf.d/xxx.conf，该目录已经被 Nginx 主配置文件（/etc/nginx/nginx.conf）的 http 模块引入。

# 3.1 配置静态资源服务

server {
	listen 80;
	server_name localhost;

	location / {
		alias /www/blog/;  # 使用alias，则/www/blog后的文件与url一一对应
	    index index.html; # 默认就是index.html, 可省略
		# autoindex on    # 可以通过目录方式访问，类似文件服务器
	}
}

# 3.2 配置反向代理服务

upstream local {
    server localhost:8080;
}

server {
    listen 80;
    server_name www.xucongcong.top; # 匹配域名（多个反向代理服务可以都使用80端口，但使用域名区分代理规则）

    location / {
		proxy_set_header Host $host;
		proxy_set_header X-Real-IP $remote_addr;
		proxy_set_header X_Forward-For $proxy_add_x_forwarded_for;
        proxy_pass http://local;
    }
}

# 3.3 配置负载均衡

upstream local {
    server localhost:8080; # 权重weight默认相同, 详细的权重配置见文档
    server localhost:8081;
}

server {
    listen 80;
    server_name www.xucongcong.top;

    location / {
        proxy_pass http://local;
    }
}

# 3.4 配置 https

先下载证书到指定目录，这里放在/etc/nginx/cert 目录下

# 这个配置能将80端口的http请求重定向到443的https请求
server {
	listen 80;
	server_name www.xucongcong.top;
	rewrite ^(.*)$ https://$host$1;
}

server {
    listen 443 ssl;
    server_name www.xucongcong.com;

    ssl_certificate cert/www.xucongcong.com.pem;
    ssl_certificate_key cert/www.xucongcong.com.key;
    ssl_session_timeout 5m;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;
    ssl_protocols TLSv1.1 TLSv1.2 TLSv1.3; #表示使用的TLS协议的类型。
    ssl_prefer_server_ciphers on;

	location / {
		alias /www/blog/;
	}
}

# 3.5 配置 websocket

server {
    listen 80;
    server_name ws.repo;

    location / {
        proxy_pass http://127.0.0.1:3000/;
        proxy_redirect off;

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
  }
}


upstream ws_server {
  server 127.0.0.1:3000;
}

server {
    listen 80;
    server_name ws.repo;

    location / {
        proxy_pass http://ws_server/;
        proxy_redirect off;

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
