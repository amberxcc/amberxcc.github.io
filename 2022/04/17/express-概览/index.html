# 1. Express 介绍

Express的最大特色，也是最重要的一个设计，就是提出中间件的概念。一个Express应用，就是由许许多多的中间件来完成的，如下图所示:

  • 请求和响应对象：request和response对象参与全程，可以被中间件改造和使用

  • 中间件：中间件的加载顺序由挂载顺序决定。

# 2. 快速开始

  1. 实例化一个express对象的实例
  2. 挂载几个简单的路由中间件
  3. 打开服务器开始监听端口，准备处理请求
const express = require('express')

const app = express()

app.get('/', (request, response)=>{
    response.send(`/:Hello, world`)
})
app.post('/', (request, response)=>{
    response.send(`/:post`)
})
app.head('/', (request, response)=>{
    response.send(`/:head`)
})
app.delete('/', (request, response)=>{
    response.send(`/:delete`)
})

app.listen(3000, ()=>{
    console.log('server start at localhost:3000')
})

# 3. 请求和响应对象

Express不对http模块已有特性进行二次抽象，只进行扩展：

  • 请求对象继承自：http.IncomingMessage() （API: Express.request (opens new window)，http.IncomingMessage (opens new window)）
  • 响应对象继承自：http.ServerResponse() （API: Express.response (opens new window)，http.ServerResponse (opens new window)）
const express = require('express')

const app = express()

app.get('/', (request, response)=>{
    response.write(request.url)
    console.log(request.url)
    console.log(request.baseUrl)
    console.log(request.headers)
    response.end()
})

app.listen(3000, ()=>{
    console.log('server start at localhost:3000')
})

# 3.1 常用属性和方法

response的常用方法：

// 一次性返回，
response.send()

// 一次性返回一个json对象
response.json()

// 多次写入再返回
response.status(201)
response.write('hello')
response.end()

response.status(201).write('hello').end('over')

# 4. 中间件分类

中间件函数可以执行以下任何任务：

  • 执行任何代码
  • 修改 request 或者 response 响应对象
  • 结束请求响应周期
  • 调用下一个中间件

调用中间件的语法：

  • next()：传给下一个中间件处理
  • next('route')：跳过当前中间件调用栈 (一个中间件可以定义的多个处理函数)，传给下一个
  • next(err)：跳到错误处理中间件处理

# 4.1 应用程序级中间件

不关心请求路径：

var app = express();

app.use(function(req, res, next) {
  console.log("Time:", Date.now());
  next();
});

app.use('/', function(req, res, next) {
  console.log("Time:", Date.now());
  next();
});

// 上面两种写法相同，不作任何匹配，所有请求都执行
// 如果use('/foo') 则匹配所有以/foo开头的请求

限定请求路径：

app.use("/user/:id", function(req, res, next) {
  console.log("Request Type:", req.method);
  next();
});

限定请求方法：

app.get("/user/:id", function(req, res, next) {
  res.send("USER");
});

多个处理函数：

app.use(
  "/user/:id",
  function(req, res, next) {
    console.log("Request URL:", req.originalUrl);
    next();
  },
  function(req, res, next) {
    console.log("Request Type:", req.method);
    next();
  }
);

多个路由处理函数：

app.get(
  "/user/:id",
  function(req, res, next) {
    console.log("ID:", req.params.id);
    next();
  },
  function(req, res, next) {
    res.send("User Info");
  }
);

// handler for the /user/:id path, which prints the user ID
app.get("/user/:id", function(req, res, next) {
  res.end(req.params.id);
});

最后一个例子：

app.get(
  "/user/:id",
  function(req, res, next) {
    // if the user ID is 0, skip to the next route
    if (req.params.id === "0") next("route");
    // otherwise pass the control to the next middleware function in this stack
    else next();
  },
  function(req, res, next) {
    // render a regular page
    res.render("regular");
  }
);

// handler for the /user/:id path, which renders a special page
app.get("/user/:id", function(req, res, next) {
  res.render("special");
});

# 4.2 路由级中间件

创建路由实例：

var router = express.Router();

示例：

var app = express();
var router = express.Router();

// a middleware function with no mount path. This code is executed for every request to the router
router.use(function(req, res, next) {
  console.log("Time:", Date.now());
  next();
});

// a middleware sub-stack shows request info for any type of HTTP request to the /user/:id path
router.use(
  "/user/:id",
  function(req, res, next) {
    console.log("Request URL:", req.originalUrl);
    next();
  },
  function(req, res, next) {
    console.log("Request Type:", req.method);
    next();
  }
);

// a middleware sub-stack that handles GET requests to the /user/:id path
router.get(
  "/user/:id",
  function(req, res, next) {
    // if the user ID is 0, skip to the next router
    if (req.params.id === "0") next("route");
    // otherwise pass control to the next middleware function in this stack
    else next();
  },
  function(req, res, next) {
    // render a regular page
    res.render("regular");
  }
);

// handler for the /user/:id path, which renders a special page
router.get("/user/:id", function(req, res, next) {
  console.log(req.params.id);
  res.render("special");
});

// mount the router on the app
app.use("/", router);

# 4.3 错误处理中间件

挂载一个错误处理中间件：中间件函数传入四个参数时，就会被当作错误处理中间件挂载。

app.use(function(err, req, res, next) {
		console.error(err.stack);
		res.status(500).send("Something broke!");
});

注意挂载顺序，前面定义的中间件可以传给后面的错误处理中间件

调用语法实例：

app.use('/', function(req, res) => {
		try{
				// ...
		}catch(err){
				next(err)
		}
})
