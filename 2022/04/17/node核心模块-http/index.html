Node核心模块—http

xucongcong
2022-04-17
  • Node

Node.js 中的HTTP接口旨在封装许多难以使用的特性。特别是大的、块编码的消息，http接口不会缓冲整个请求或响应，因此用户能够流式传输数据。


# 1. http 模块的结构与功能

为了支持所有可能的 HTTP 应用程序，Node.js HTTP API 是非常底层的。它只进行流处理和消息解析。它只将消息解析为headers 和 body，但不解析headers 和 body的详细内容。如接收到的原始 headers 保留在 rawHeaders 属性中，该属性是 [key, value, key2, value2, ...] 的数组：

[ 'ConTent-Length', '123456',
  'content-LENGTH', '123',
  'content-type', 'text/plain',
  'CONNECTION', 'keep-alive',
  'Host', 'example.com',
  'accepT', '*/*' ]

# 1.1 http模块的主要功能

  • 作为服务端：创建一个 http server 用于处理http请求
  • 作为客户端：创建一个 http client 用于请求服务端的数据

# 1.2 http模块的的构成

  • 核心类
    • http.Agent：负责管理 HTTP 客户端连接的持久性和重用
    • http.ClientRequest：负责管理 HTTP 客户端请求
    • http.Server：负责管理 HTTP 服务端
    • http.ServerResponse：负责管理 HTTP 服务端响应
    • http.IncomingMessage：用于访问响应状态、headers和数据
    • http.OutgoingMessage：对传出消息的抽象
  • 核心API
    • http.createServer()：创建一个HTTP Server
    • http.request()：创建一个HTTP Client
    • http.get()：创建一个请求方法为GET的HTTP Client

http 与 https 模块: 这两个模块都是相似的，但是当我们创建 https server时，需要提供安全证书，而对于 http，则不需要。


# 2. 实现一个 HTTP 客户端

使用 https 模块向某个页面发出请求：

const https = require('https')
const fs = require('fs')

const options = {
    hostname: 'www.xucongcong.top',
    port: 443,
    path: '/2020/01/01/helloworld/',
    method: 'GET'
}

const request = https.request(options, (res) => {
    let responseBody = ''
    console.log('response from server started');
    console.log(`Server status: ${res.statusCode}`)

    // 响应是一个流对象，可以监听事件
    res.on('data', (chunk) => {
        console.log(`--chunk length--${chunk.length}`)
        responseBody += chunk
    })
    res.on('end', () => {
        fs.writeFile('hello-blog.html', responseBody, (err) => {
            if (err) { throw err; }
            console.log('File is downloaded');
        })
    })
})

// 要查看是否有任何错误，侦听 request 对象上的error事件
request.on('error', (err) => {
    console.log('error:', err.message);
})

// 最后需要手动结束我们的请求，否则会超时
request.end()

# 3. 实现一个HTTP服务端

http.createServer([options][, requestListener])

requestListener 是自动添加到 'request' 事件的函数。

实现一个普通的 http server：

const http = require('http')

const server = http.creatServer((req, res)=>{
	res.writeHead(200,{'Content-Type': 'text/plain'})
	res.end('hello world')
})

server.listen(3000, ()=>console.log('server start at 3000'))

如果是https服务需要提供证书：

const https = require('http')
const fs = require('fs')

// 类型一
const options = {
	key: fs.readFileSync('.../agent2-key.pem'),
	cert: fs.readFileSync('.../agent2-cert.pem')
}
// 类型二
const options = {
	pfx: fs.readFileSync('./test_cert.pfx'), 
  passphrase:'sample'
}

// 添加证书选项
const server = https.creatServer(options, (req, res)=>{
	res.writeHead(200,{'Content-Type': 'text/plain'})
	res.end('hello world')
})

server.listen(3000, ()=>
	console.log('server start at 3000')
)

# 4. 实现一个文件服务器

  • 建立一个public文件夹
  • 文件夹里面存放html和jpg文件
const http = require('http')
const fs = require('fs')
const path = require('path')

const server = http.createServer((req, res) => {
    if (req.url === '/') {
        fs.readFile("./public/default.html", "UTF-8", function (err, html) {
            res.writeHead(200, { "Content-Type": "text/html" })
            res.end(html)
        })
    } else if (req.url.match(/.jpg$/)) {    // 匹配host:port/xxx.jpg
        const imgPath = path.join(__dirname, 'public', req.url)
        const imgStream = fs.createReadStream(imgPath)
        res.writeHead(200, { "Content-Type": "image/jpeg" })
        imgStream.pipe(res)
    } else {
        res.writeHead(404, { "Content-Type": "text/plain" })
        res.end("404 Not Found")
    }
})

server.listen(3000, () =>
    console.log('server start at 3000')
)
