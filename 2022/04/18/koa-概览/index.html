Node Web Framework—Koa

xucongcong
2022-04-18
  • Koa
  • Node

# 1. 概述

利用HTTP模块我们已经能构建 Web应用，但是这个模块还是过于底层，使用起来是比较的繁琐，最早对HTTP模块进行封装的框架中比较流行的一个有 Express, 它做了很多简化的工作和扩展增强，最主要是引进了中间件的概念, 获得了很多好评， Express 就是 koa 的前生. 也可以这么看Koa是ES6版的Express，原班人马, 只是把ES5语法改成ES6，并且换了个名，目前新的Koa版本是 Koa2。

Koa没有在其核心中捆绑任何中间件。经过几年的发展，已经有丰富的插件扩展，甚至还有在它的基础上又封装了一层的框架，例如阿里的 Egg.js。


# 2. 快速开始

const koa = require('koa')

const app = new koa()

app.use(ctx => {
    ctx.body = 'Hello koa'
})

app.listen(3000, ()=>{
    console.log('start at 3000')
})

# 3. Context（上下文）

Koa Context 将 node 的 request 和 response 对象封装在一个单独的对象里面。context在每个 request 请求中被创建，在中间件中接收使用：

app.use(async ctx => {
    ctx;          // is the Context
    ctx.req;       // http原生的requset对象
    ctx.res       // http原生的response对象
    ctx.request;  // koa封装的Request对象
    ctx.response; // koa封装的Response对象
});

许多 context 的访问器和方法为了便于访问和调用，简单的委托给他们的 ctx.request 和 ctx.response 所对应的等价方法， 比如说 ctx.type 和 ctx.length 代理了 response 对象中对应的方法，ctx.path 和 ctx.method 代理了 request 对象中对应的方法。（详见API文档）


# 4. 路由

koa本身没有内置路由系统，原生的路由写法是：

app.use(async ctx => {
    if(ctx.path === '/test'){
        ctx.body = 'test'
    }
    else ctx.body = 'Home'
});

要获得express风格的路由系统，需要引入其他中间件：

const Koa = require('koa')
const Router = require('koa-router')

const app = new Koa()
const router = new Router()

router.get('/', ctx => {
    ctx.body = 'Home'
})

router.get('/test', ctx => {
    ctx.body = 'test'
})

app.use(router.routes())
app.use(router.allowedMethods())

app.listen(3000, ()=>{
    console.log('start at 3000')
})

# 5. 中间件

koa中间件是洋葱模型，每个中间件的调用像是一个栈结构，在请求进入和出来时都会被调用一次。

const koa = require('koa')

const app = new koa()

app.use((ctx,next) => {
    console.log('1<=')
    next()
    console.log('=>1')
    ctx.body = "hello"
})
app.use((ctx,next) => {
    console.log('2<=')
    next()
    console.log('=>2')
})
app.use((ctx,next) => {
    console.log('3<=')
    next()
    console.log('=>3')
})
app.listen(3000, ()=>{
    console.log('start at 3000')
})

调用next()会传递执行权，执行完后可以继续执行next()后的代码。

💡 如果没有手动调用next，执行权不会往后传递


# 6. 错误处理


# 6.1 单个中间件处理错误

app.use(async ctx => {
    if(ctx.path === '/test') ctx.body = 'test'
	else if(ctx.path === '/500') ctx.thow(500)
    else if(ctx.path === '/404') ctx.thow(404)
});

# 6.2 注册最外层中间价为所有中间件处理错误

const koa = require('koa');
const { isNamedExportBindings } = require('typescript');

const app = new koa()

app.use(async (ctx, next) => {
    try{
        await next()
    }catch(err){
        ctx.body = err.message
    }
})

app.use(async (ctx, next) => {
    if(ctx.path === '/'){
        ctx.body = 'home'
    }else{
        await next()
    }
})

app.use(async (ctx, next) => {
    if(ctx.path === '/test'){
        throw new Error('test error')
        ctx.body = 'test'
    }
    await next()
})

app.listen(3000, ()=>{
    console.log('start at 3000')
})

💡 建议所有函数都用async，然后await next()，不然同步函数无法传递异步执行的错误


# 6.3 通过监听error事件处理错误

如果错误未被手动捕获，则会触发error事件：

app.on('error', (err, ctx)=>{
	  console.log(err)
})

错误若被手动捕获则不会触发error事件，也可以在捕获后手动触发：

try{
	...
	ctx.app.emit('error', err, ctx) // 可以但没必要，一般在顶层手动捕获处理
}
