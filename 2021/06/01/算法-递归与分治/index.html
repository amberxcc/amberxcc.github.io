# 1. 递归（Recursion）

💡 递归 <—> 循环 （循环调用自己）

【递归代码模板】

# 参数：当前层数、传到当前层的变量
def recursion(level, param1, param2, ...):

    # 1. 设定递归终止条件
    if level > MAX_LEVEL:
        return given_result
            
    # 2. 处理当前层的数据
    process_data(level, param1, param2, ...)

    # 3. 把数据传递给下一层
    recursion(level+1, p1, p2)

    # 4. 递归回来后要做的收尾工作(可选)
    reverse_state(level)

示例一：计算阶乘

# 套模板实现版本
def Factorial(n):
		# 1. 设定递归终止条件：n<=1时指定返回值
    if n<=1 :
			return 1

    # 2. 处理当前层的数据：不需要特殊处理，直接拿本层层数
    cur_level_data = n

    # 3. 把数据传递给下一层：不需要额外的参数，只传递层数
    next_level_data = Factorial(n-1)

    # 4. 递归回来后返回本层的处理结果
    return cur_level_data * next_level_data

#---------------------------------------------------
# 简化实现版本
def Factorial(n):
    if n<=1:
		return 1

	return n * Factorial(n-1)
#---------------------------------------------------

示例二：斐波那契数列

def fib(n):
	if n==0 or n==1:
		return 1

	return fib(n-1)+fib(n-2)

💡 类似这种问题虽然可以递归解决，但很多子问题被重复计算，浪费了性能


# 2. 分治（Divide & Conquer）

把一个大问题分解为不同的子问题，子问题一一解决再返回（互不相关的子问题可以并行执行），如果子问题之间相互依赖则不使用分治，而是使用动态规划或记录子问题结果等办法。

【分治代码模板】

def divide_conquer(problem, param1, param2, ...):

		# 1. 指定递归终止条件
		if problem is None:
			print given_result
			return given_result

		# 2. 准备数据、分解问题
		data = prepare_data(problem)
		subproblem = split_problem(problem, data)

		# 3. 递归处理子问题
		subresult1 = divide_conquer(subproblem[0], p1, ...)
		subresult2 = divide_conquer(subproblem[1], p1, ...)
		subresult3 = divide_conquer(subproblem[2], p1, ...)
		...

		# 4. 整合子问题的处理结果
		result = process_result(subresult1,subresult2,subresult3, ...)
