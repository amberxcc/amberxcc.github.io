Git—本地基础操作

xucongcong
2021-02-21
  • Git

# 1. 初始化（init）

命令 含义
git init 初始化仓库，创建.git文件夹管理文件

该命令将创建一个名为 .git 的子目录，这个子目录含有初始化的 Git 仓库中所有的必须文件，这些文件是 Git 仓库的骨干。 但是，在这个时候，仅仅是做了一个初始化的操作，项目里的文件还没有被跟踪。


# 2. 提交到暂存区（add）

命令 含义
git add 将工作区文件提交到暂存区
git add . 批量提交到暂存区

# 3. 提交到本地仓库（commit）

命令 含义
git commit -m "..." 提交暂存区到本地仓库，并附带信息
git commit -am "..." 把所有跟踪的文件改动一并add到暂存区，然后提交
git commit -amend -m "..." 修改（替换）最后一次的提交信息
💡 如果新增了一个文件，那么必须要用 git add 命令跟踪新文件，再用 git commit -m 提交。

如果只是修改了已跟踪的文件，那么可以用 git commit -am 命令一起完成“提交文件到暂存区、提交暂存区的文件”（省略了 git add 命令，因为修改文件也必须add才能commit）。


# 4. 状态（status）

命令 含义
git status 命令输出仓库当前的状态
git status -s/--short 格式更为紧凑的输出

新添加的未跟踪文件前面有 ?? 标记，新添加到暂存区中的文件前面有 A 标记，修改过的文件前面有 M 标记。 输出中有两栏，左栏指明了暂存区的状态，右栏指明了工作区的状态。


# 5. 比较差异（diff）

命令 含义
git diff 比较工作区与暂存区
git diff --cached(staged) 比较暂存区与与上一次commit
git diff HEAD 比较工作区与上一次commit
git diff HEAD^ 比较工作区与上上次commit
git diff HEAD^^ 比较工作区与上上上次commit
git diff HEAD~2 比较工作区与上上上次commit
git diff <commitId> 比较工作区与指定 commitid

# 6. 版本回退（reset）

命令 含义
git reset [HEAD] 重置当前分支的HEAD为指定commit，同时重置暂存区，但工作区不变
git reset [HEAD] [--] files 取消对某个文件的暂存
git reset --hard HEAD 重置当前分支的HEAD为指定commit，同时重置暂存区和工作区
git reset --keep HEAD 重置当前分支的HEAD为指定commit，同时保持暂存区和工作区不变

# 7. 删除和移动（rm，mv）

命令 含义
git rm file 将文件从暂存区和工作区中删除
git rm --cached file 从暂存区中删除，但不删除工作目录中的文件
git rm -f file 强行从暂存区和工作区中删除修改后的文件
git mv [file] [newfile] 移动或重命名一个文件、目录或软连接
git mv -f [file] [newfile] 新文件名已经存在，但还是要重命名它

运行 git mv 就相当于运行了下面三条命令：

$ mv README.md README
$ git rm README.md
$ git add README
💡 如此分开操作，Git 也会意识到这是一次重命名，所以不管何种方式结果都一样。 两者唯一的区别在于，`git mv` 是一条命令而非三条，直接使用 `git mv` 方便得多。 不过在使用其他工具重命名文件时，记得在提交前 `git rm` 删除旧文件名，再 `git add` 添加新文件名。

# 8. 日志（log）

命令 含义
git log 查看 commit 提交历史
git log --pretty=oneline 简化log输出内容
git log --grapg 图形化显示提交记录
git reflog 查看git命令的历史记录

# 9. 取消跟踪（gitignore）

.gitignore中指定的的文件不会被跟踪

# 忽略所有的 .a 文件
*.a

# 但跟踪所有的 lib.a，即便你在前面忽略了 .a 文件
!lib.a

# 只忽略当前目录下的 TODO 文件，而不忽略 subdir/TODO
/TODO

# 忽略任何目录下名为 build 的文件夹
build/

# 忽略 doc/notes.txt，但不忽略 doc/server/arch.txt
doc/*.txt

# 忽略 doc/ 目录及其所有子目录下的 .pdf 文件
doc/**/*.pdf

GitHub 有一个十分详细的针对数十种项目及语言的 .gitignore文件列表， 可以在 https://github.com/github/gitignore (opens new window) 找到。
