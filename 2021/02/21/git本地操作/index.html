Git-本地操作

xucongcong
2021-02-21
  • Git

# 1. 初始化（init）

命令 含义
git init 初始化仓库，创建.git文件夹管理文件

# 2. 提交到暂存区（add）

命令 含义
git add 将工作区文件提交到暂存区
git add . 批量提交到暂存区

# 3. 提交到本地仓库（commit）

命令 含义
git commit -m "..." 提交暂存区到本地仓库，并附带信息
git commit -am "..." 把所有跟踪的文件改动一并add到暂存区，然后提交
git commit -amend -m "..." 修改最后一次的提交信息

💡 如果新增了一个文件，那么必须要用 git add 命令跟踪新文件，再用 git commit -m 提交; 如果只是修改了已跟踪的文件，那么可以用 git commit -am 命令一起完成“提交文件到暂存区、提交暂存区的文件”（省略了 git add 命令，因为修改文件也必须add才能commit）。


# 4. 状态（status）

命令 含义
git status 命令输出仓库当前的状态
git status -s/--short 格式更为紧凑的输出

# 5. 比较差异（diff）

命令 含义
git diff 比较工作区与暂存区
git diff --cached 比较暂存区与与上一次commit
git diff HEAD 比较工作区与上一次commit
git diff HEAD^ 比较工作区与上上次commit
git diff HEAD^^ 比较工作区与上上上次commit
git diff HEAD~2 比较工作区与上上上次commit
git diff <commitid> 比较工作区与指定 commitid

# 6. 版本回退（reset）

git reset [--soft | --mixed | --hard] [HEAD]
1

  • --mixed

    为默认参数，用于 。

  • --hard 撤销工作区中所有未提交的修改，将暂存区与工作区都回到上一次版本，并删除之前的所有提交

  • --soft 用于回退到某个版本


# 7. 删除和移动（rm，mv）

命令 含义
git mv src dest 移动文件（移动到当前目录可用于改名）
git rm file 同时删除版本库和工作目录中的文件
git rm --cached file 从版本库中删除，但不删除工作目录中的文件

# 8. 日志（log）

命令 含义
git log 查看 commit 提交历史
git log --pretty=oneline 简化log输出内容
git reflog 查看git命令的历史记录

# 9. 取消跟踪（gitignore）

.gitignore中指定的的文件不会被跟踪。GitHub有一个十分详细的针对数十种项目及语言的.gitignore列表，查看 (opens new window)

# 忽略所有的 .a 文件
*.a

# 但跟踪所有的 lib.a，即便你在前面忽略了 .a 文件
!lib.a

# 只忽略当前目录下的 TODO 文件，而不忽略 subdir/TODO
/TODO

# 忽略任何目录下名为 build 的文件夹
build/

# 忽略 doc/notes.txt，但不忽略 doc/server/arch.txt
doc/*.txt

# 忽略 doc/ 目录及其所有子目录下的 .pdf 文件
doc/**/*.pdf
