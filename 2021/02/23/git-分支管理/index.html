Git—分支管理

xucongcong
2021-02-23
  • Git

# 1. git branch（分支管理）

$ git branch             # 列出所有分支，当前分支前面会标一个*号

$ git branch testing     # 从当前HEAD创建一个新的分支

$ git branch -d <分支名>  # 删除（已合并的）分支

$ git branch -D <分支名>  # 强制删除分支

# 2. git switch （分支切换）

$ git switch master  # 尝试切换到已有的master分支
$ git checkout master

$ git switch -c dev  # 创建并切换到新的dev分支
$ git checkout -b dev

切换分支这个动作，用switch更科学


# 3. git checkout（检出）

checkout命令用于从历史提交（或暂存区）中拷贝文件到工作目录（也可用于切换分支）。

从指定的提交中拷贝指定文件（覆盖）到暂存区域和工作目录。

$ git checkout <commitId> files

给出一个（本地）分支时，可以切换分支。新提交节点（下图中的a47c3）中的所有文件都会被复制到暂存区域和工作目录中；只存在于老的提交节点（ed489）中的文件会被删除；不属于上述两者的文件会被忽略，不受影响。

$ git checkout <分支名>

# 4. git merge（合并分支）

合并指定分支到当前分支

$ git merge <分支名>
💡 合并分支可能产生冲突，冲突需要手动选择，然后再提交。
1. git branch（分支管理）
2. git switch （分支切换）
3. git checkout（检出）
