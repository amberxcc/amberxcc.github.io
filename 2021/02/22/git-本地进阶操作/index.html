Git—本地进阶操作

xucongcong
2021-02-22
  • Git

# 1. git stash （暂存修改）

$ git stash  # 将暂存区未提交的内容缓存起来

$ git stash save "1号缓存"  # 为暂存添加标记信息

$ git stash list  # 查看stash中的所有暂存信息

$ git stash pop  # 弹出栈顶的第一个暂存内容

$ git stash pop stash@{1} # 弹出一个指定的暂存内容

# 2. git tag （打标签）

$ git tag  # 查看所有标签

$ git tag -a "v1.0" -m "发布1.0版本"  # 给最近一次提交的版本打一个标签

$ git tag -a "v0.9" f8ec6 -m "发布0.9版本"  # 历史上的某个版本打一个标签

$ git tag -d v0.9  # 删除某个标签

# 3. git rebase（变基）

场景一：合并多条提交记录，简化log（建议只合并未push的commit）

$ git rebase -i <commitid>
  1. 选择合并的提交
  2. 编辑合并后的commit信息

场景二：将merge后的分支信息，合并到合并后的分支（取舍）

  1. 分别把hotfix和v2提交commit
  2. 切换到hotfix所在的dev分支，执行git rebase master
  3. 切换到v2所在的master分支，执行git merge dev

场景三：取代使用fetch和rebase取代pull，简化log

  1. git fetch origin branchname
  2. git rebase origin/branchname
  3. 如果有冲突，解决冲突、git add .
  4. git rebase —continue

😄 注意：若rebase过程产生冲突，手动解决冲突后，git add . 再git rebase —continue
