MongoDB—简介

xucongcong
2021-05-04
  • MongoDB

# 1. 为什么用MongoDB？

MongoDB是最近十年间成熟起来的数据库，在其之前无一例外的是有着30多年历史的关系型数据库产品，作为一个新时代的成熟数据库产品，，MongoDB的优势在于：

  • 对于开发者：MongoDB使用了类似对象模型的JSON数据结构，大幅度简化了数据库模式设计、ORM层编码的工作，从而可以更专注于业务
  • 对于DBA：MongoDB可以解决系统高可用、高性能、横向扩展问题
  • 对于架构或产品：MongoDB是一个非常灵活的数据库，可以快速响应业务需求的变化

# 2. MongoDB与SQL

  • 文档是 MongoDB 中的基本数据单元，可以粗略地认为其相当于关系数据库管理系统中的行。
  • 集合可以被看作具有动态模式的表。
  • 一个MongoDB实例也可以拥有多个独立的数据库，每个数据库都拥有自己的集合。
  • 每个文档都有一个特殊的键 "_id"，其在所属的集合中是唯一的。

# 3. MongoDB数据类型

MongoDB的数据类型采用了类似JSON的BSON存储模型，BSON 是一种二进制序列化格式，用于在 MongoDB 中存储文档和进行远程过程调用。 BSON 规范位于 bsonspec.org (opens new window)。

每种 BSON 类型都有整数和字符串标识符，如下表所示：
